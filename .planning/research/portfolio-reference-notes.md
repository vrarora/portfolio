# Portfolio Reference Notes

**Sources:** `https://www.oriol.design/` extracted with `designlang` on 2026-04-27, and `The UX Case Study Crisis_ Breaking Free from Templates.pdf`.

## Oriol.Design Design Language

- Visual target: sparse editorial portfolio with large negative space, compact floating navigation, oversized black display typography, thin pill CTAs, flat cards, and scattered asymmetric section placement.
- Hero pattern: small credibility line with avatar/status, huge uppercase role statement, two compact CTAs, supporting client/team marks, and scroll cue.
- Content pattern: service cards, selected work as concise proof points, principles cards, about/certifications, writing/speaking, testimonials.
- Palette: mostly `#ffffff`, `#000000`, `#0000ee`, `#808080`, `#5b5b5b`, with sharp accents like `#fd431d`, `#ffb800`, `#02ab4e`.
- Typography: extracted families include Aeonik, Mabry Pro, Times, and sans-serif; practical implementation should use licensed-safe alternatives unless the user provides font files.
- Motion: minimal, mostly 300ms cubic-bezier transitions and scroll-linked reveal behavior; initial loader should be skipped per user request.
- Accessibility: extraction reported 100% WCAG color-pair score; preserve strong contrast and keyboard navigability.

## Case Study Strategy From PDF

- Avoid template-driven case studies that list generic process steps without showing why each step mattered.
- Optimize for hiring managers who scan quickly: the work must communicate value in 30 seconds, then reward deeper reading.
- Lead with context, role, constraints, final output, and impact before expanding into process.
- Structure each case study around the designer's point of view, decisions, tradeoffs, and insights rather than a fixed checklist.
- Use scannable headings that tell the story even if the reader only skims headings.
- Show polished visuals and only include artifacts that strengthen the argument; recreate messy deliverables if needed.
- Make the writing personal: why this work mattered, what the designer believes, where the judgment was non-obvious.

## Product Direction Hypothesis

Build a product designer portfolio inspired by Oriol.Design's interaction model and layout language, but using the user's own identity, work, copy, visuals, and case-study framing. Treat the reference as a design-language target, not a source for copied proprietary content.

## Rachel Chen Case Study Structure Reference

Source inspected: `https://www.rachelchen.tech/projects/openai` on 2026-04-27.

- Detail page structure: back link, top navigation, in-page section index, project eyebrow, large title, metadata columns for role/timeline/team/skills, then long-form scrollytelling.
- Narrative order: overview → problem → opportunity → solution → core flows → research → exploration → prototyping/testing → insights → design decisions → constraints/tradeoffs → reflection.
- Strong pattern: every major section has a sharp headline before supporting copy and visuals.
- Good fit for this portfolio: use Rachel-style case-study detail pages, but visually restyle them with the Oriol-inspired homepage language so the whole site feels cohesive.
